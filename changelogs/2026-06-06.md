# 2026-06-06 变更记录

## [refactor][build][docs][BREAKING] 将 selector 迁移到 tynavi

### 变更内容
- 移除项目内自有 `Selector` 运行时实现与 `onebot-api-macros` 中的 `Selector` derive 宏实现。
- 新增可选依赖 `tynavi`，并将 `selector` feature 改为启用 `tynavi` 的 derive 能力。
- 将事件类型与消息段类型的 selector 派生迁移为 `tynavi::Selector`，并用 tynavi 的 `route_*`、`*_filter`、`is_*` 风格替代旧 API。
- 保留 `onebot_api::selector` 作为 tynavi selector 类型与 traits 的再导出入口，降低调用方迁移成本。
- 更新 README 与 TODO 中的 selector 文档，移除旧 `.selector()`、`and_filter_*`、`indexof()` 等示例。

### 涉及文件
- `Cargo.toml` [Changed] 新增 `tynavi` 可选依赖，并让 `selector` feature 启用该依赖。
- `src/lib.rs` [Changed] 移除本地 selector 模块导出，改为再导出 tynavi 及其 selector traits。
- `src/selector.rs` [Removed] 删除旧版本地 `Selector` 与 `AsSelector` 实现。
- `macros/src/lib.rs` [Changed] 移除 `#[derive(Selector)]` 过程宏入口，仅保留 API sender 宏。
- `macros/src/selector.rs` [Removed] 删除旧版 Selector derive 宏实现。
- `macros/docs/selector.md` [Removed] 删除已废弃的本地 Selector derive 宏文档。
- `src/event.rs` [Changed] 将顶层事件 selector 派生迁移到 tynavi，并移除旧 `through` 属性。
- `src/event/message.rs` [Changed] 将消息事件 selector 派生迁移到 tynavi，并收紧 quick_operation 相关导入的 feature gate。
- `src/event/meta.rs` [Changed] 将 meta 事件 selector 派生迁移到 tynavi。
- `src/event/notice.rs` [Changed] 将 notice 事件 selector 派生迁移到 tynavi。
- `src/event/request.rs` [Changed] 将 request 事件 selector 派生迁移到 tynavi，并收紧 quick_operation 相关导入的 feature gate。
- `src/message/receive_segment.rs` [Changed] 将接收消息段 selector 派生迁移到 tynavi，并避免与 `EnumIs` 重复生成 `is_*` 方法。
- `src/message/utils.rs` [Changed] 将 `AtType` selector 派生迁移到 tynavi，并避免与 `EnumIs` 重复生成 `is_*` 方法。
- `README.md` [Changed] 将 selector 章节改为 tynavi API 示例。
- `TODO_LIST.md` [Changed] 将 selector 测试待办调整为 tynavi 接入后的验证方向。
- `changelogs/2026-06-06.md` [Added] 新增本次 selector 迁移记录。

### 影响
- 启用 `selector` feature 的调用方需要从旧 `.selector()`、`and_filter_*`、`*_selector()`、`indexof()` 等 API 迁移到 tynavi 的 `as_selector()`、`route_*()`、`*_filter()`、`extract()` / `extract_async()` 等 API。
- `onebot-api-macros` 不再提供 `Selector` derive 宏；需要派生 selector 的代码应使用 `tynavi::Selector`。
- 未启用 `selector` feature 时，原有 `EnumIs` 判断方法仍保持可用；启用后由 tynavi 生成同名 `is_*` 方法。

### 原因
- 将 selector 能力迁移到独立的 `tynavi` crate，减少 onebot-api 内部宏和运行时选择器维护成本。
- 统一使用 tynavi 的不可变 selector 模型，为后续父节点回溯、路由和过滤能力复用打基础。
