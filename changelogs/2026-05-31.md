# 2026-05-31 变更记录

## 移除 anyhow，全面使用 thiserror 自定义错误

### 变更内容
- 移除了主 crate `Cargo.toml` 中的 `anyhow` 依赖（`macros/` 与测试中的 `anyhow` 保留）。
- 在 `src/error.rs` 中扩展 `ServiceRuntimeError` 与 `ServiceStartError` 枚举，新增以下变体：
  - `ServiceRuntimeError::ChannelClosed` — 内部 channel 关闭
  - `ServiceRuntimeError::SerializeError(#[from] serde_json::Error)` — 序列化/反序列化失败
  - `ServiceRuntimeError::WebSocketClosedByPeer` — WebSocket 被对端关闭
  - `ServiceRuntimeError::WebSocketError` — WebSocket 错误
  - `ServiceRuntimeError::WebSocketStreamEnded` — WebSocket 流结束
  - `ServiceRuntimeError::EventSourceEnded` — SSE EventSource 结束
  - `ServiceRuntimeError::UrlCannotBeBase` — URL 无法添加 path segment
  - `ServiceRuntimeError::ReqwestError(#[from] reqwest::Error)` — HTTP 请求错误
  - `ServiceStartError::InvalidSecretLength(#[from] hmac::digest::InvalidLength)` — HMAC secret 长度非法
- 将 `src/communication/` 下各模块的 `anyhow::Result` 统一替换为 `ServiceRuntimeResult` 或 `ServiceStartResult`：
  - `src/communication/utils.rs`
  - `src/communication/ws_reverse.rs`
  - `src/communication/http.rs`
  - `src/communication/http_post.rs`
  - `src/communication/sse.rs`
  - `src/communication/combiner.rs`

### 影响
- 生产代码不再依赖 `anyhow`，错误类型更加明确，便于调用方做针对性处理。
- 保留 `macros/` 与示例中的 `anyhow`，不影响编译期宏和测试代码。
